1. QA - Qaulity assurance, Prevents the bugs, process oriented 
QC - Qaulity control, find the bugs, product oriented
verification - software meets desgin, to test the design, reviewing code, documentation, by developer
validation - software meets end user, to test the functionality of product, maintainence, by tester

2. risk based testing - testing the product/ software using previous experience, common mistakes, usual bugs
risk = impact x likelihood
when time and resource are limited

3. Severity - Impact of the bug on the functionality
priority - How soon the bug should be fixed
high S , high p - a mobile device is crashed while outgoing call
low S, low p - spelling mistake in about section

4. Good Tc - TC ID, Objective, Pre condition, input data, test steps, expected, actual, req ID
I create test scenarios from reuiments or user stories, then I map those scnarios to req, then I create test cases covering the scenarios and req

5. EP - to divide the the data into various parts, only secific data can be tested instead of testing all the data (population of city, tier 1(10 lakhs+), 2(5 lakh to 10 lakhs) etc)
BVA - to test only the boundry input and out values from given data (voter id age above 18 only)

Feature - Create user profile
Scenarios - 1. User is able to create profile
	2. User is able to enter 3 to 20 charaters only in username field
	3. valid email format
	4. Phone should have 10 digits only
	5. Age should be between 18 to 99
	6. Bio can have max 500 characters
	7. After submitting, profile is created successfully
Test case - 
	TC_01 : To verify user can create a profile
	Pre-condition : 1. New user
	Steps : 1. Click on create profile
	Expected : User is able to land on new page with profile details on it

	TC_02 : To verify that user is able to enter only 3 characters in username field
	Pre-condition : new user is on create profile page
	Steps : 1. Click on username field
		2. Enter any 3 letters in username field
	data : abc
	Expected : letters abc should be successfully entered on username

	TC_03 : To verify that user is able to enter only 20 characters in username field
	Pre-condition : new user is on create profile page
	Steps : 1. Click on username field
		2. Enter any 20 letters in username field
	data : abcdqwertyasdfghzxcvb
	Expected : letters abcdqwertyasdfghzxcvb should be successfully entered on username

	TC_04 : To verify that user is not able to enter 2 characters in username field
	Pre-condition : new user is on create profile page
	Steps : 1. Click on username field
		2. Enter any 2 letters in username field
	data : ab
	Expected : letters ab should be entered on username but not acceptable by username field

	TC_05 : To verify that user is not able to enter 21 characters in username field
	Pre-condition : new user is on create profile page
	Steps : 1. Click on username field
		2. Enter any 21 letters in username field
	data : abcdqwertyasdfghzxcvbq
	Expected : Those 21 letters should be entered on username not acceptable by username field

	TC_06 : To verify that user is able to enter valid email format in email field
	Pre-condition : new user is on create profile page
	Steps : 1. Click on email field
		2. Enter valid email in email field
	data : abc@gmail.com
	Expected : email should be successfully entered in email field
